## 📁 **PureAudio — Структура проекта**

|------|-----------|

### 🖼 **View (WPF Windows)**
| Файл | Назначение |
|------|-----------|

| `Views/HelpWindow.xaml` / `.xaml.cs` | Окно справки |

### 🧠 **ViewModels (MVVM)**
| Файл | Назначение |
|------|-----------|
| `ViewModels/MainViewModel.cs` | **Главная VM** — связывает AudioService, PlaylistService, FFT, управление состоянием плеера, тосты, busy-индикатор |
| `ViewModels/ExpandedPanelViewModel.cs` | VM для нижней панели (библиотека + плейлист) — навигация, поиск, сканирование, импорт/экспорт |

### 🎵 **Audio Engine (Services)**
| Файл | Назначение |
|------|-----------|
| `Services/AudioService.cs` | **Центральный аудио-сервис** — Shared/Exclusive режимы, Bit Perfect логика, fallback, управление воспроизведением |
| `Services/WasapiExclusivePlayer.cs` | **Кастомный IWavePlayer** для WASAPI Exclusive (низкоуровневый, Bit Perfect) |
| `Services/DeviceCapabilities.cs` | Определение возможностей DAC (макс. sample rate, bit depth, channels), проверка Bit Perfect статуса |
| `Services/BitPerfectWaveProvider.cs` *(внутри AudioService)* | Провайдер для Exclusive режима — читает raw PCM, обходит ISampleProvider |
| `Services/FlacReader.cs` | Нативный FLAC-декодер (IWaveProvider) для Bit Perfect цепи |
| `Services/SoxResampler.cs` | Ресемплинг через SoX (fallback, когда формат трека превышает возможности DAC) |
| `Services/FftService.cs` | Вычисление FFT (спектр) — логарифмические полосы, пики |
| `Services/FftSampleProvider.cs` | ISampleProvider, который перехватывает float-сэмплы для FFT |
| `Services/FftQueue.cs` | Потокобезопасная очередь FFT-данных |

## 🔍 **Ключевые точки для рефакторинга**

### 1. **AudioService.cs (~850+ строк) — главный кандидат**
Содержит:
- `BitPerfectWaveProvider` (внутренний класс)
- Управление Shared/Exclusive режимами
- Position tracking (таймер)
- Volume control
- Cue-треки
- FFT-интеграцию

**Возможные проблемы**: слишком много ответственности, смешана логика воспроизведения, статуса и FFT.

### 2. **MainViewModel.cs (~500+ строк)**
Содержит:
- Связывание всех сервисов
- Управление UI-состоянием (цвета, текст, иконки)
- Toast / busy-индикатор
- Marquee-текст
- FFT timer

**Возможные проблемы**: много логики отображения, можно вынести в отдельные "контроллеры состояния".

### 3. **ExpandedPanelViewModel.cs (~500+ строк)**
Содержит:
- Логику библиотеки (сканирование, поиск, фильтрация)
- Логику плейлиста (up/down/delete/clear)
- Импорт .m3u/.pls
- Работу с диалогами

**Возможные проблемы**: смешана работа с библиотекой и плейлистом.

### 4. **LibraryService.cs (~500+ строк)**
Содержит:
- Сканирование файловой системы
- Построение дерева
- Кэширование (JSON)
- Hi-Res / MP3 разделение

**Возможные проблемы**: кэширование и сканирование можно разделить.

### 5. **DeviceCapabilities.cs**
Содержит:
- WASAPI probe
- Format matching
- Bit Perfect status

**Статус**: относительно компактный и сфокусированный.

## 💡 **Рекомендации по рефакторингу (на выбор)**

| Приоритет | Что рефакторить | Почему |
|-----------|----------------|--------|
| 🔴 **High** | `AudioService.cs` | Разделить на `PlaybackEngine` (воспроизведение) + `AudioStateManager` (статусы) + вынести `BitPerfectWaveProvider` в отдельный файл |
| 🟡 **Medium** | `MainViewModel.cs` | Вынести "статус-менеджер" (цвета, тексты, индикаторы) в отдельный `StatusDisplayController` |
| 🟡 **Medium** | `ExpandedPanelViewModel.cs` | Разделить на `LibraryPanelViewModel` + `PlaylistPanelViewModel` |
| 🟢 **Low** | `LibraryService.cs` | Вынести кэширование в отдельный `LibraryCacheService` |
| 🟢 **Low** | `FftService.cs` + `FftQueue.cs` + `FftSampleProvider.cs` | Объединить в один `FftPipeline` или оставить как есть (они и так неплохо разделены) |
